What is a Software Forge?

A software forge is a centralized place for developers to control and manage software development.

The core element of any software forge is a repository for files managed with a version control tool. Many projects limit the function of their forge to version control; the term forge would be meaningless to those projects.

A fullfledged collaborative development environment typically provides more than version control:

  • a home page
  • pages for:
    • release management
    • software download packages
    • defect and issue tracking
    • documents
    • forums
    • support
  • defect and issue tracking
  • site administration
Any project needs these capabilities regardless of the degree of integration.

Software forges are not inherently restricted to software. Any activity generating work products in the form of computer files (such as, CAD/CAM drawings) could manage those files with the tools of a software forge.

Forge Technology

The team is currently experimenting with various version control tools:

  • CVS — the Concurrent Versions System
  • Subversion, intended as a replacement for CVS

Employing a collaborative development environment can dictate a particular version control tool. Such environments include:

  • SourceForge
  • GForge

Cost is an issue when considering collaborative development environments.
