Phoenix Project
Systems Architecture and Engineering

Project Description

The purpose of this project is:

  • to elicit system requirements for the Phoenix Ambulatory Blood Pressure System
  • to devise an architecture that satisfies those requirements and that is buildable under an open-source model

Systems engineering is not typically applied to open-source products but is appropriate to the Phoenix System, which fits the conditions of a complex system:

An interacting combination of elements to accomplish a defined objective. These include:
  • Hardware (the monitor)
  • Software (analysis applications)
  • Firmware (monitor OS, monitor applications)
  • People (clinical and laboratory roles)
  • Information
  • Techniques (clinical protocols)
  • Facilities (the Halberg Chronobiology Center)
  • Services
  • Other support elements
(International Council on System Engineering, System Engineering Handbook)

Project Deliverables

The deliverables are to be structured and organized so that they align with the other Phoenix Projects:

  • System requirements specification
  • Architectural design description
