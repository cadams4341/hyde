Phoenix Ambulatory Blood Pressure Monitor Project
Public Works of Invention, Public Disclosure and Public Invention Release

 

We are placing the intellectual property of the Phoenix Project into the public domain so that others can use it and we can obtain the benefits of collaboration, and to establish a universal platform for measuring blood pressure cycles and applying the information to diagnosis and treatment.

Our intellectual property can be categorized as hardware, software and non-software/hardware copyrights and trademarks. Hardware is usually copyrighted and often patented, software is usually copyrighted, and other items are typically trademarked or copyrighted. A summary of pubic hardware licenses can be found at www.opencollector.org/hardlicense.

Phoenix Public Domain Non-Software or Hardware Process

  • Label each document (including webpages) with the Phoenix Release, as shown in our "How to Label" FAQ.
  • For software copyrights, follow the Open Source GPL guidelines at www.gnu.org/licenses/gpl.html
  • For hardware copyrights, follow the Open Source 'HardLicense' guidelines at www.opencollector.org/hardlicense/hdpl.html

Phoenix Public Domain Software Process

  • Label each document (including webpages) with the Phoenix Release.
  • For the final document, follow the Open Source GPL guidelines at www.gnu.org/licenses/gpl.html

Phoenix Public Domain Hardware Process

  • Label each document (including webpages) with the Phoenix Release.
  • For hardware copyrights, follow the Open Source 'Hardware License' guidelines at http://www.opencollector.org/hardlicense/hdpl.html
  • When the 'method' is final, use the United States Statutory Invention Registration. It is exactly like a patent, except it is only used for defensive purposes. For an example, go to: www.uspto.gov and search the patent database for number 'H1,444' .
  • Complete the process of placing the patent into the public domain with the Public Invention Release.
